Emergency Access Only — Break-Glass Protocol
Break-glass access bypasses normal RBAC controls and grants elevated production access to tenant data planes. This mechanism is strictly for emergencies (P1 incidents, data recovery, security response). All sessions require manager approval, are time-limited, fully recorded, and subject to mandatory post-incident review. Unauthorized use triggers immediate security escalation.
Request Break-Glass Access
Required. Must reference an active P1/P2 incident.
Approval Workflow
Break-glass access follows a strict multi-stage approval pipeline. Each stage is logged and immutable.
1
Requested
Operator submits break-glass request with justification and incident ref
2
Manager Approval
Designated manager reviews and approves/rejects within 15 min SLA
3
Security Review
Automated scope check + security team notified. Grants least-privilege access.
4
Active Session
Full session recording. All queries and mutations logged with hash chain.
5
Auto-Expired
Session auto-terminates at duration limit. Post-incident review required within 48h.
